# Техническая документация Zelda GeoGuessr

## 📖 Общее описание

Zelda GeoGuessr – это веб-игра на чистом JavaScript, использующая карту в формате одного изображения (без тайлов) для простоты развёртывания. Основная цель – угадать местоположение по скриншоту.

## 🧩 Структура проекта

```
ZeldaGeoGuessr/
├── index.html           # Главная страница
├── style.css            # Стили
├── locations_data.js    # Данные о сериях и локациях
├── js/
│   ├── config.js
│   ├── scoring.js       # Расчёт расстояния и очков
│   ├── map.js           # Инициализация и работа Leaflet
│   ├── ui.js            # Управление экранами, HUD, просмотрщик изображений
│   └── game.js          # Основная игровая логика
├── map/
│   └── full_map.jpg     # Карта 18000×15000 px (108 МБ)
└── locs/                # Скриншоты локаций (PNG)
```

## 🗺️ Карта и координаты

- **Размер**: 18 000 × 15 000 пикселей.
- **Система координат**: начало (0,0) – левый верхний угол, X растёт вправо, Y – вниз.
- **Привязка к Leaflet**: используется `L.CRS.Simple`; точка (x, y) преобразуется в `L.latLng(-y, x)`.  
  Это сделано для того, чтобы ось Y на карте шла сверху вниз, как в пиксельной системе.

### Ключевые функции в `map.js`

- `pxToLatLng(x, y)` – преобразование в Leaflet-координаты.
- `latLngToPx(latlng)` – обратное преобразование.
- `create(containerId)` – создание карты с изображением-оверлеем.
- `createGuessMarker(latlng)` – красный маркер ответа игрока.
- `createCorrectMarker(latlng)` – зелёный маркер правильного ответа.
- `createResultLine(from, to)` – жёлтая линия между ними.

## 📊 Данные локаций (`locations_data.js`)

Файл содержит массив `series`. Каждая серия:

```js
{
  "id": 1,
  "name": "Название серии",
  "rounds": [
    { "image": "имя_файла.png", "x": 4500, "y": 3800 },
    ...
  ]
}
```

- `image` – имя файла в папке `locs/`.
- `x`, `y` – координаты правильного ответа в пикселях карты.

## 🧮 Подсчёт очков

Функция `Scoring.score(distance)` использует формулу:

```js
score = Math.round( MAX_ROUND_SCORE * Math.exp(-distance / CALIBRATION_D) )
```

- `MAX_ROUND_SCORE = 5000`
- `CALIBRATION_D = 2000` (подбирается эмпирически)

Расстояние вычисляется по Евклидовой метрике:  
`sqrt((x1-x2)^2 + (y1-y2)^2)`.

## 🖼️ Просмотрщик изображений

Реализован в `ui.js` в объекте `UI`. Позволяет:

- Увеличивать/уменьшать изображение колёсиком мыши или кнопками.
- Перетаскивать (drag) для панорамирования.
- Сбрасывать масштаб и центрирование двойным кликом или кнопкой «⟲».
- Закрывать по `Escape` или кнопке «✕».

## 🧪 Планируемые улучшения

1. **Режим отладки**  
   Добавить в `config.js` флаг `DEBUG: true`. При его включении:
   - на карте отображаются координаты клика;
   - в результатах раунда показывается калибровочное значение расстояния.

2. **Поведение просмотрщика**  
   - При открытии изображение должно центрироваться по центру вьюпорта (сейчас – по левому верхнему углу).
   - Сброс масштаба должен приводить к полному центрированию, а не только к исходному `fit`.
   - Зум должен происходить строго относительно положения курсора (уже реализовано частично, но требует доработки).

3. **Динамическая калибровка**  
   Возможность менять `CALIBRATION_D` через интерфейс для тестирования.

## 📦 Зависимости

- [Leaflet](https://leafletjs.com/) v1.9.4 – картографическая библиотека.
- Шрифты, стили – нативные.

## 🧑‍💻 Разработка

Все модули собраны в глобальные объекты (`UI`, `GameMap`, `Scoring`, `CONFIG`), чтобы упростить доступ.  
При добавлении новых серий просто расширьте `locations_data.js`. Новые скриншоты кладите в папку `locs`.
